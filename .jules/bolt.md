## 2025-05-24 - N+1 Insert Optimization
**Learning:** `createMany` is supported in Prisma for SQLite and is significantly faster than sequential `create` calls in a loop. However, `createMany` does not return the created records, which can be a limitation if the UI expects the immediate result. `prisma.$transaction([...creates])` is a robust alternative that batches operations (atomicity) and returns results, though it may not be as performant as `createMany` for raw insertion speed, it solves the N+1 round-trip latency issue partially and guarantees data integrity.
**Action:** When bulk inserting data where the result is needed immediately (e.g., syncing external data to local DB and showing it), use `prisma.$transaction(items.map(i => prisma.model.create(i)))`. If results are not needed, prefer `prisma.model.createMany({ data: items })`.
